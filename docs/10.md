**🤖 AI 分析**:
### OpenClaw v2026.3.7 版本发布分析

#### 主要新功能概述
1. **Agents/context 引擎插件接口**：新增 `ContextEngine` 插件槽，支持完整的生命周期钩子（如 `bootstrap`, `ingest`, `assemble` 等），并提供基于配置的解析和 `LegacyContextEngine` 包装器，以保留现有的压缩行为。这使得插件可以提供替代的上下文管理策略，而无需修改核心压缩逻辑。
2. **ACP/持久化通道绑定**：增加了对 Discord 通道和 Telegram 话题绑定的持久化存储、路由解析和 CLI/文档支持，确保 ACP 线程目标在重启后仍然有效，并且可以一致地管理。
3. **Telegram/ACP 话题绑定**：支持在 `/acp spawn` 命令中使用 Telegram Mac Unicode 虚线选项前缀，支持 Telegram 话题线程绑定（`--thread here|auto`），并将绑定确认消息固定在话题内。
4. **Web UI/i18n**：添加了西班牙语 (`es`) 本地化支持，包括语言检测、延迟加载和语言选择器标签。
5. **Onboarding/web 搜索**：在配置向导中添加了提供商选择步骤和完整的提供商列表，并支持在引导过程中使用 SecretRef 引用模式。
6. **Docker/Podman 扩展依赖项烘焙**：新增 `OPENCLAW_EXTENSIONS`，允许容器构建时预安装选定的扩展 npm 依赖项，从而加快和提高容器部署的可重复性。

#### 重要改进点
1. **Gateway 认证模式**：当同时配置了 `gateway.auth.token` 和 `gateway.auth.password` 时，现在需要显式设置 `gateway.auth.mode` 为 `token` 或 `password`，以避免启动和配对失败。
2. **TTS/OpenAI 兼容端点**：新增 `messages.tts.openai.baseUrl` 配置支持，具有配置优先于环境变量的优先级，并支持 OpenAI TTS 请求路由到解析的基础 URL。
3. **Slack/DM 打字反馈**：在 Socket Mode DM 中添加了反应式的处理状态显示，即使 Slack 本地助手打字不可用时也能显示处理状态。

#### 对用户的影响
1. 新版本通过引入新的插件接口和持久化通道绑定，极大地增强了系统的灵活性和稳定性，使得用户可以更方便地管理和扩展其 AI 助手的功能。
2. 通过增加多语言支持和改进的认证机制，新版本显著提升了用户体验，特别是在多渠道集成和国际化的场景下。

5. **Telegram 直接消息主题**：增加了每个直接消息的主题配置（允许列表、策略、技能等），并路由主题作为独立的会话，增强了授权和防抖动机制。

3. **Cron 任务轻量级启动上下文**：添加了轻量级启动模式选项，减少了自动化运行时的启动文件注入。
