## 🏗️ OpenClaw 架构概览

### 核心理念

OpenClaw 采用 **分布式 + 本地优先** 的架构设计，核心理念是：

- **隐私优先**: 所有数据存储在本地，不上传云端
- **多渠道集成**: 统一管理各种消息渠道（WhatsApp、Telegram、Slack 等）
- **可扩展性**: 插件化技能系统，轻松扩展功能
- **AI 无处不在**: 将 AI 能力集成到日常使用的各种渠道中

### 架构组件

```
┌─────────────────────────────────────────────────────────┐
│                     用户消息渠道                          │
│  WhatsApp | Telegram | Slack | Discord | iMessage | ...  │
└──────────────────────┬──────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────┐
│                   Gateway (控制平面)                     │

└──────────────────────┬──────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────┐
│                   ClawdHub (技能注册表)                  │
│            内置技能 + 社区贡献的第三方技能                │
└─────────────────────────────────────────────────────────┘
```

### 核心组件说明

#### 1. Gateway（网关）- 控制平面
Gateway 是 OpenClaw 的核心大脑，负责：
- 🧠 **会话管理**: 维护所有对话的状态和上下文
- 🔄 **消息路由**: 智能分发消息到正确的处理器
- ⚡ **技能调度**: 根据用户意图调用相应技能
- 🤖 **AI 集成**: 与 Anthropic、OpenAI、xAI 等 LLM 交互
- 💾 **数据持久化**: 本地 SQLite 数据库存储

**技术栈**: TypeScript + Node.js/Bun + SQLite + WebSocket

#### 2. Node（渠道节点）
Node 是运行在各设备上的轻量级代理：
- 📱 **macOS Node**: 支持 iMessage、系统通知
- 📱 **iOS Node**: 快捷指令集成
- 🤖 **Android Node**: Android 服务集成
- 💻 **Linux Node**: 命令行工具 + 系统服务

#### 3. Skills（技能系统）
插件化的功能扩展机制：
- 🔍 **内置技能**: 搜索、计算、翻译、天气
- 🔧 **工具技能**: 调用外部 API 和服务
- 🧠 **AI 技能**: 基于 LLM 的智能任务处理
- 📦 **社区技能**: ClawdHub 提供丰富的第三方技能

### 典型工作流程

```
用户发送消息（WhatsApp）
    ↓
Node 接收并转发
    ↓
Gateway 解析意图
    ↓
调用 AI 模型 + 执行技能
    ↓
生成响应
    ↓
返回到 WhatsApp
```
